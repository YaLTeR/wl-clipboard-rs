Perform lazy binding.

Relocations shall be performed at an implementation-defined time, ranging from the time of the Library::open call until the first reference to a given symbol occurs. Specifying RTLD_LAZY should improve performance on implementations supporting dynamic symbol binding since a process might not reference all of the symbols in an executable object file. And, for systems supporting dynamic symbol resolution for normal process execution, this behaviour mimics the normal handling of process execution.

Conflicts with RTLD_NOW.
